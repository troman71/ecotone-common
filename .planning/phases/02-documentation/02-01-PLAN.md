
5. **Passwords module** (`ecotone_common.passwords`)
   - `hash_password(password: str) -> str` — hash with bcrypt, returns UTF-8 string
   - `check_password(password: str, hashed: str) -> bool` — verify against bcrypt hash
   - `validate_strength(password, min_length=8, require_upper=True, require_lower=True, require_digit=True) -> list[str]` — returns list of failure messages (empty = valid)
   - Code example showing all three functions

6. **Tokens module** (`ecotone_common.tokens`)
   - `TokenService(secret_key: str)` — constructor, takes any secret string
   - Generic: `generate_token(data: dict, salt: str) -> str` and `validate_token(token: str, salt: str, max_age: int = 3600) -> dict | None`
   - Email verification: `generate_verification_token(email: str) -> str` and `validate_verification_token(token: str, max_age: int = 3600) -> str | None`
   - Password reset: `generate_reset_token(email: str) -> str` and `validate_reset_token(token: str, max_age: int = 3600) -> str | None`
   - Approval: `generate_approval_token(user_id, action: str) -> str` and `validate_approval_token(token: str, max_age: int = 86400) -> dict | None`
   - Invite: `TokenService.generate_invite_token() -> str` (static, random) and `TokenService.validate_invite_token(token: str) -> bool` (static, format check only — DB lookup is caller's responsibility)
   - Code example showing password reset flow (generate + validate)

7. **Email module** (`ecotone_common.email`)
   - `create_email_backend(config: dict) -> EmailBackend` — factory function
   - Config keys checked in order: `SENDGRID_API_KEY` → SendGridBackend, `SMTP_HOST` → SmtpBackend, otherwise → LogBackend
   - `backend.send(to: str, subject: str, html_body: str) -> bool`
   - Code example showing dev mode (LogBackend via empty config dict) and production (SendGrid config dict)
   - List the config keys: SENDGRID_API_KEY, FROM_EMAIL, FROM_NAME, SMTP_HOST, SMTP_PORT, SMTP_USERNAME, SMTP_PASSWORD

8. **Consent module** (`ecotone_common.consent`)
   - `log_consent(cursor, user_id, consent_type: str, version: str, accepted: bool, ip_address: str = None, user_agent: str = None)` — inserts into consent_log table
   - `get_current_eula_version(cursor) -> str | None` — returns latest version or None
   - Note: requires `consent_log` and `eula_versions` tables in the caller's database
   - Code example showing EULA acceptance flow
